1. INTRODUCTION

Health informatics, document analysis, and artificial learning systems have grown rapidly, allowing automation of complex analysis. Similarly, legal document analysis—particularly Terms and Conditions (TnC) and Privacy Policies—presents a significant challenge to everyday users. These agreements are famously long, obfuscated, and packed with dense legal jargon.

The TnC-Bot is an advanced artificial learning and health/legal informatics system that helps users instantly read, parse, query, and comprehend digital contracts. Through structured extraction, contextual embeddings, and an interactive Retrieval-Augmented Generation (RAG) system, it translates dense legal agreements into plain-English clauses under key domains.

1.1 Problem Statement

A Terms and Conditions agreement is a legal contract that users must accept to use a service. However, survival of user rights and data privacy is difficult to predict because these agreements are uncommon in plain formatting and vary significantly in structure. The research work carried out uses deep learning language models and custom vector store pipelines (built from scratch) to detect and highlight sensitive clauses in scanned or digital agreements, helping users instantly identify data rights, AI training permissions, and hidden liabilities.

1.2 Existing System

A contract analysis system is developed using basic keyword classification techniques. In the existing system, researchers use various machine learning approaches to classify text clauses using static patterns. Also, many classifiers have been used in the literature, such as SVM and Naïve Bayes classifiers. Consequently, these systems are constrained to predefined classes and cannot interpret custom uploaded documents dynamically or answer user queries grounded in real-time document context.

1.3 Proposed System

The proposed model is introduced to overcome the disadvantages that arise in the existing system. The threshold parsing process provides the advantage of extracting text blocks from scanned image/PDF scan files. The classification and grounded retrieval using a local Vector database and a Large Language Model are applied. Finally, the grounded RAG algorithm is used to predict answers based on the contract text, evaluating precision, recall, and contextual similarity.

1.4 Scope

This project aims to revolutionize user rights protection by leveraging informatics systems, natural language processing, and advanced text mining techniques. The focus is on implementing sophisticated document processing methods to analyze legal agreements, extracting crucial privacy insights. Utilizing diverse local databases, our initiative seeks to enhance the accuracy and readability of privacy policies, contributing to ongoing efforts in securing consumer rights online.

3.2 Implementation

Python is the pool of libraries. It has numerous libraries for real-world applications. One such library is PyMuPDF. PyMuPDF is a cross-platform library used for PDF and document processing. It includes applications like layout rendering and text extraction. It is majorly used in text transformations, document scanning, metadata extraction, and other informatics applications.

The steps involved in document analysis and grounded chat are:

  1. Importing the required modules: Load FastAPI, SQLite, fitz (PyMuPDF), NumPy, and OpenAI.
  2. Building the Ingestion Flow: Setup endpoints to accept pasted text or uploaded binary streams.
  3. Document OCR Parsing: Fallback to PyTesseract image-to-string extraction if PDF text layers are empty.
  4. Paragraph-Aware Chunking: Implement split logic to divide policies into overlapping text blocks.
  5. Vector Similarity Engine: Write cosine similarity calculations to search local document chunks.
  6. Structured Information Completion: Instruct language models to return JSON-compliant summary metadata.
  7. Explanation Translation Styles: Setup endpoints to translate chosen snippets into Simple, Teen, Technical, or Legal terms.
  8. Grounded Stream Processing: Pipe matching vector chunks into LLM prompts and stream responses to clients.
  9. Chrome Overlay Integration: Build manifest configurations to scrap active elements and display findings.

Step 1: Document Upload & PDF Text Ingestion:

In this initial step, the code performs essential text extraction on the input PDF. The file stream is passed through PyMuPDF. If no text layers are found, the document is rasterized into temporary images and processed using PyTesseract OCR to retrieve the text buffer.

Step 2: Sentence-Aware Chunking:

Splits the document text into smaller, overlapping chunks (1000 characters with 200 character overlap). Iteratively looks for paragraph breaks or sentence terminals so that statements do not lose context mid-sentence.

Step 3: SQLite Vector Store & Mapping:

Sends chunks to the OpenAI embeddings API (or fallback parameters). Writes chunk text alongside coordinate arrays to tnc_bot.db. If offline or credentials are missing, maps search terms using Jaccard keyword overlap calculations.

Step 4: Structured Data Completion:

Sends the initial contract text block to generative completion endpoints requesting compliance with a strict JSON format. The output yields the agreement category, a summary, and lists of key clauses.

Step 5: Explanation Style Translation:

Receives a specific selected text snippet and translates it according to user selection. Integrates instructions for Simple (8th-grade level), Teen-Friendly (analogies), Technical (data policies), or Legal (indemnity limits) styles.

Step 6: Grounded Chatbot Interaction:

Accepts user questions in the chatbot tab, executes a similarity search on the SQLite index, retrieves the top 4 matched blocks, and injects them as prompt context. The chatbot streams responses citing specific blocks (e.g. [Block 1]).

Step 7: Browser Overlay Content Script:

The Chrome extension content script runs inside client browser windows. It scrapes text content from active policy webpages, submits the payload to the local FastAPI router, and displays highlights inside a floating panel on the page.

# vector_store.py: Cosine Similarity Vector Search Loop
import numpy as np

def search_similarity(openai_client, doc_id: str, query: str, top_k: int = 4):
    chunks = get_document_chunks(doc_id)
    if not chunks or not chunks[0].get("embedding"):
        return keyword_fallback_search(chunks, query, top_k)
        
    query_emb = get_embedding(openai_client, query)
    query_vector = np.array(query_emb)
    
    scored_chunks = []
    for chunk in chunks:
        chunk_vector = np.array(chunk["embedding"])
        
        # Cosine Similarity: (A . B) / (||A|| * ||B||)
        dot_product = np.dot(query_vector, chunk_vector)
        norm_a = np.linalg.norm(query_vector)
        norm_b = np.linalg.norm(chunk_vector)
        
        similarity = dot_product / (norm_a * norm_b) if norm_a > 0 and norm_b > 0 else 0
        scored_chunks.append((similarity, chunk))
        
    scored_chunks.sort(key=lambda x: x[0], reverse=True)
    
    results = []
    for score, chunk in scored_chunks[:top_k]:
        chunk_copy = chunk.copy()
        chunk_copy["score"] = float(score)
        if "embedding" in chunk_copy:
            del chunk_copy["embedding"]
        results.append(chunk_copy)
        
    return results
# routes/analysis.py: Document Analysis Pipeline
from fastapi import APIRouter, UploadFile, File
import uuid

router = APIRouter()

@router.post("/analyze/pdf")
async def analyze_pdf(file: UploadFile = File(...), category: str = "Uncategorized"):
    pdf_bytes = await file.read()
    
    # Extract text using PyMuPDF
    text, needs_ocr = extract_text_from_pdf_bytes(pdf_bytes)
    
    # Fallback to OCR if scanned
    if needs_ocr:
        text = run_ocr_on_pdf_fallback(pdf_bytes)
        
    # Index document blocks in vector database
    doc_id = str(uuid.uuid4())
    save_document(doc_id, file.filename, text, calculate_hash(text), category)
    index_document(doc_id, text)
    
    # Retrieve structured analysis
    summary_data = analyze_agreement_instant(text)
    update_document_summary(doc_id, summary_data)
    
    return {
        "id": doc_id,
        "name": file.filename,
        "content": text,
        "summary": summary_data
      }
